# ThingsBoard-Bot Documentation 🤖

> **AI-Powered Orchestrator for Real-Time IoT Intelligence**

This document provides a comprehensive technical overview of the ThingsBoard-Bot project, designed for developers and architects.

## 🏗️ 1. Architecture Overview

The system uses a **Layered N-Tier Architecture** integrated with a **Context-Augmented Generation (CAG)** workflow. It acts as an intelligent, stateless orchestration proxy between end-users and the ThingsBoard IoT platform.

### **Core Design Principles:**
*   **Identity Scoping:** All data fetches are performed using the end-user's JWT token, ensuring strict RBAC (Role-Based Access Control).
*   **Logic Decoupling:** Sub-systems (CCTV, FAS, IAS, etc.) are treated as independent logic entities.
*   **Mathematical Truth:** Device counts and critical statuses are calculated in Java and forced upon the AI via `SYSTEM_NOTE` to prevent hallucinations.
*   **Professional Clarity:** Output is strictly formatted for readability using a "Bold Answer + Small Reason" structure.

## 🚀 2. Key Features

### **Full Data Context**
The backend performs a 100% fetch of all telemetry and attributes for every branch, providing the AI with the complete state of the network.

### **Sub-System Independence**
The bot independently monitors 6 core sub-systems per branch:
1. **CCTV** (`cctv`)
2. **Integrated Alarm System (IAS)** (`ias`)
3. **Burglar Alarm System (BAS)** (`bas`)
4. **Fire Alarm System (FAS)** (`fas`)
5. **Time Lock System (TLS)** (`timeLock`)
6. **Access Control System (ACS)** (`accessControl`)

### **Definitive Offline Logic**
A branch is classified as **OFFLINE** if its gateway status is "Offline", "Fault", "N/A", or "Inactive".

### **Key Normalization**
Automatically maps technical ThingsBoard attributes (e.g., `cctvStatus`, `iasStatus`) to simplified keys used by the AI.

## 🔄 3. Request Lifecycle (Data Flow)

1.  **Ingress:** User asks a question via the floating chat widget.
2.  **Authentication:** `ChatController` extracts the user's JWT from the `X-TB-Token` header.
3.  **Cache Check:** `UserDataService` checks for valid data (5-minute TTL).
4.  **Full Fetch:** If empty, `UserAwareThingsBoardClient` retrieves 100% of data from ThingsBoard.
5.  **Normalization:** `ContextFilterUtil` cleans the data and maps technical keys to business keys.
6.  **Pruning:** If context > 10,000 tokens, `ChatService` performs keyword-based pruning to keep relevant data.
7.  **Inference:** `ChatService` injects a calculated `SYSTEM_NOTE` and history into the OpenAI prompt.
8.  **Egress:** AI generates a professional, non-truncated response.

## 📂 4. Component Breakdown

### **Service Layer (Logic)**
*   **`ChatService.java`**: The "Heart." Manages orchestration, pruning, and AI rules.
*   **`UserDataService.java`**: The "Identity Manager." Handles per-user caching and secure data scoping.
*   **`ChatMemoryService.java`**: The "Short-term Memory." Manages a sliding window of the last 4 messages.

### **Integration Layer (API Clients)**
*   **`UserAwareThingsBoardClient.java`**: High-performance client using user JWTs. Includes JSON flattening for complex telemetry.
*   **`OpenAIClient.java`**: Specialized wrapper for GPT-4o-mini communication.

### **Utility Layer**
*   **`ContextFilterUtil.java`**: Strict whitelist manager and key normalizer.
*   **`TokenCounterService.java`**: Real-time token usage estimator.

## 🧠 5. AI Prompt Engineering

The bot is governed by a strict `SYSTEM_PROMPT` located in `ChatService.java`.

### **Mandatory Rules:**
*   **Strict Privacy:** Refuse vague requests (e.g., "Give me info on X") and ask for specific categories.
*   **Branch-Centric:** Use "Branch" instead of "Device" and prioritize `branchName` over technical IDs.
*   **Zero means No:** If alarm counts are 0, explicitly state "No active alarms."
*   **Output Structure:** 
    *   **Bold Summary Line** at the top.
    *   *Italicized Technical Reason* at the bottom.
    *   Bulleted lists for all multi-item responses.

### **Key Environment Variables:**

*   `TB_URL`: Base URL of your ThingsBoard instance.

### **System Limits (application.properties):**

*   `max-tokens`: 2,000 (AI answer length).

## 📝 7. Developer Onboarding
To add a new sub-system:
1.  Define the attribute in `ContextFilterUtil.java` (IMPORTANT_KEYS).
2.  Add normalization mapping in `ContextFilterUtil.java`.
3.  Update the **Sub-Device Indicators** list in `ChatService.SYSTEM_PROMPT`.
4.  Add the key to the keyword-pruning list in `ChatService.java`.
