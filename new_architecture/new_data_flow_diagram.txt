TARGET FLOW (v4.0 — Event-Driven)

  ┌─────────────────────────────────────────────────────────────────────────┐
  │                    THINGSBOARD PRIVATE CLOUD PE                         │
  │                                                                         │
  │  [6,000 IoT Devices] ──MQTT──▶ [TB Internal Kafka (50GB)]               │
  │                                                                         │
  │  [Rule Engine — 3 Chains]                                               │
  │                                                                         │
  │  Chain 1: State Change Detector                                         │
  │    Filter: only pass if value actually changed                          │
  │                                                                         │
  │  Chain 2: Payload Formatter (TBEL)                                      │
  │    msg.tb_message_id  = metadata.msgId;         // UUID                 │
  │    msg.customer_id    = metadata.customerId;    // "BOI"                │
  │    msg.branch_node_id = metadata.branchNodeId;  // "node_boi_br042"     │
  │    msg.log_type       = determineLogType(...);                          │
  │    msg.field          = mapLogTypeToField(...);                         │
  │    msg.new_value      = msg[msg.field];                                 │
  │    msg.prev_value     = metadata.prevValue;                             │
  │    msg.event_time     = metadata.ts;                                    │
  │                                                                         │
  │  Chain 3: Webhook Dispatcher                                            │
  │    POST /webhooks/tb                                                    │
  │    Header: X-HMAC-SHA256: {signature}                                   │
  │    Retry: RETRY_ALL, 3x, 5s backoff                                     │
  └──────────────────────────────┬──────────────────────────────────────────┘
                                 │
                                 │  HTTP POST + HMAC
                                 ↓
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                    WEBHOOK RECEIVER (Spring Boot)                       │
  │                                                                         │
  │  1. Validate HMAC-SHA256                     (< 0.5ms)                  │
  │  2. Parse TbEventPayload                                                │
  │  3. Push to in-memory buffer (BlockingDeque, cap 10,000)                │
  │  4. Return HTTP 200                          (< 5ms total)              │
  │                                                                         │
  │  Background thread drains buffer ──▶ RabbitMQ                           │
  │    queue: iot.{customer_id}.events                                      │
  │    (e.g. iot.BOI.events, iot.SBI.events)                                │
  │                                                                         │
  │  JVM shutdown hook: drain buffer with 5s timeout                        │
  └──────────────────────────────┬──────────────────────────────────────────┘
                                 │
                                 │  RabbitMQ (per-customer quorum queue)
                                 ↓
  ┌─────────────────────────────────────────────────────────────────────────┐
  │                    EVENT CONSUMER (Spring Boot)                         │
  │                                                                         │
  │  @RabbitListener(queues = "iot.{customerId}.events")                    │
  │                                                                         │
  │  ┌──────────────────────────────────────────────────┐                   │
  │  │ Step 1: IDEMPOTENCY CHECK                        │                   │
  │  │   INSERT into device_events                      │                   │
  │  │   (tb_message_id UUID UNIQUE constraint)         │                   │
  │  │   Duplicate? → ACK + discard                     │                   │
  │  └──────────────────────┬───────────────────────────┘                   │
  │                         │                                               │
  │  ┌──────────────────────▼───────────────────────────┐                   │
  │  │ Step 2: TIMESCALEDB WRITE (system of record)     │                   │
  │  │   device_events table (hypertable)               │                   │
  │  │   customer_id, branch_node_id, field,            │                   │
  │  │   prev_value, new_value, event_time              │                   │
  │  │   DB failure → NACK (RabbitMQ retries)           │                   │
  │  └──────────────────────┬───────────────────────────┘                   │
  │                         │                                               │
  │  ┌──────────────────────▼───────────────────────────┐                   │
  │  │ Step 3: LOAD ANCESTOR PATH (< 1ms, from cache)   │                   │
  │  │   AncestorPathCache.getAncestors(                │                   │
  │  │     "BOI", "node_boi_br042")                     │                   │
  │  │   → ["node_boi_ho",                              │                   │
  │  │      "node_boi_fgmo_north",                      │                   │
  │  │      "node_boi_zo_kolkata"]                      │                   │
  │  │   (length varies: 2 for IOB, 4 for SBI)          │                   │
  │  └──────────────────────┬───────────────────────────┘                   │
  │                         │                                               │
  │  ┌──────────────────────▼───────────────────────────┐                   │
  │  │ Step 4: EXECUTE LUA SCRIPT (Redis, atomic)       │                   │
  │  │                                                  │                   │
  │  │   ARGV[1] = "BOI"          (customer_id)         │                   │
  │  │   ARGV[2] = "node_boi_br042" (branch_node_id)    │                   │
  │  │   ARGV[3] = "gateway_status" (field)             │                   │
  │  │   ARGV[4] = "ONLINE"       (new_value)           │                   │
  │  │   ARGV[5] = "OFFLINE"      (prev_value)          │                   │
  │  │   ARGV[6] = 1712400000     (epoch)               │                   │
  │  │   ARGV[7] = "node_boi_ho"  ← ancestor 1          │                   │
  │  │   ARGV[8] = "node_boi_fgmo_north" ← ancestor 2   │                   │
  │  │   ARGV[9] = "node_boi_zo_kolkata"  ← ancestor 3  │                   │
  │  │                                                  │                   │
  │  │   Lua atomically:                                │                   │
  │  │   a) HSET BOI:branch:state:node_boi_br042        │                   │
  │  │        gateway_status = ONLINE                   │                   │
  │  │        last_updated = 1712400000                 │                   │
  │  │   b) HINCRBY BOI:global:counters                 │                   │
  │  │        total_OFFLINE -1                          │                   │
  │  │        total_ONLINE +1                           │                   │
  │  │   c) FOR EACH ancestor:                          │                   │
  │  │        HINCRBY BOI:node:counters:{node_id}       │                   │
  │  │          total_OFFLINE -1                        │                   │
  │  │          total_ONLINE +1                         │                   │
  │  └──────────────────────┬───────────────────────────┘                   │
  │                         │                                               │
  │  ┌──────────────────────▼───────────────────────────┐                   │
  │  │ Step 5: ACK RabbitMQ                             │                   │
  │  │   Only after BOTH DB write AND Redis Lua succeed │                   │
  │  └──────────────────────────────────────────────────┘                   │
  └─────────────────────────────────────────────────────────────────────────┘
                                 │
                                 │  Redis now has current state
                                 │  for ALL branches, ALL customers
